### 技术细节

#### 模块依赖关系
```
config.js (配置中心)
    ↓
    ├── courses.js (使用 config.coursesDir)
    ├── data.js (使用 config.folderDataPath)
    ├── submissions.js (使用 config.classroomLayoutPath)
    ├── proxy.js (使用 config.*)
    └── socket.js (使用 config.*)
```

#### 数据流
```
客户端请求 → Express 中间件 → API 路由 → 业务模块 → 数据层
                ↓                    ↓
           代理处理            Socket.io
                ↓
           CDN/文件系统
```

## [之前版本] - 多班级机房视图支持

### 新增功能

#### 机房视图多班级管理
- ✨ 支持创建和管理多个班级的座位表
- ✨ 班级选择器快速切换班级
- ✨ 每个班级独立存储座位布局、讲台位置等配置
- ✨ 记住最后使用的班级，下次打开自动切换

#### 班级管理功能
- ✨ 新建班级：为不同班级创建独立的座位表
- ✨ 删除班级：支持删除不需要的班级（至少保留一个）
- ✨ 导出班级：导出单个班级的配置为 JSON 文件
- ✨ 导入班级：支持导入 v2 格式的班级配置文件
- ✨ 兼容旧版本：自动迁移旧版本的单表数据到"默认班级"

#### 用户体验优化
- ✨ 文件名优化：导出 JSON 文件名包含班级名称（如：`classroom-高一1班-20240324-1430.json`）
- ✨ 导入选项：导入 v2 格式 JSON 时可选择创建新班级或覆盖当前班级
- ✨ 座位数量显示：班级列表中显示每个班级的座位数量

### 改进

#### 文档更新
- 📝 更新 `README.md`，添加多班级管理功能说明
- 📝 更新 `docs/用户说明-教师端.md`，详细说明多班级使用方法
- 📝 更新 `docs/API.md`，说明学生名称与当前班级的关联
- 📝 更新 `docs/LumeSync_参赛全套文档.md`，更新功能描述

### 技术细节

#### 数据存储
- 存储键从 `classroom-layout-v1` 升级为 `classroom-layouts-v1`
- 数据结构从数组改为对象：`{ [classroomId]: { name, seats, podiumAtTop } }`
- 新增 `classroom-last-used` 键记录最后使用的班级 ID

#### 兼容性
- ✅ 自动检测并迁移旧版本数据
- ✅ 支持导入旧版本的 CSV/JSON 文件
- ✅ 导出新的 v2 格式包含班级元数据

### 使用场景

#### 一个机房服务于多个班级
- 为每个班级（如高一1班、高一2班）创建独立的座位表
- 为不同班级设置不同的座位布局和学生姓名
- 上课时快速切换到对应班级
- 学生端提交的文件名会自动使用当前班级中的学生姓名

#### 历史记录保留
- 保存每个班级的历史座位安排
- 方便后续复用
